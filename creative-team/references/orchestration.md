# Creative Team Orchestration Guide

## Purpose
This skill coordinates the design, research, storytelling, inclusion, and prompt-engineering skills into one structured creative process.

## Team Topology
### Strategy Layer
- `brand-guardian`
- `ux-researcher`

### Structure Layer
- `ux-architect`

### Craft Layer
- `ui-designer`
- `visual-storyteller`

### Safety And Polish Layer
- `inclusive-visuals`
- `image-prompt-engineer`
- `whimsy-injector`

## Routing Rules
Use the smallest team that can answer the request.

### If the request is strategic
Start with `brand-guardian`, then validate with `ux-researcher`.

### If the request is about user flow or hierarchy
Start with `ux-architect`, then refine with `ui-designer`.

### If the request is about a hero or landing-page concept
Use `brand-guardian`, `ux-researcher`, `ux-architect`, `ui-designer`, and `visual-storyteller` together.

### If the request includes imagery or prompts
Add `inclusive-visuals` before `image-prompt-engineer`.

### If the request needs personality or delight
Apply `whimsy-injector` after utility, clarity, and performance are already solved.

## Parallel Work Template
1. Brief normalization by the orchestrator.
2. Parallel exploration by selected specialists.
3. Independent critique on mobile, accessibility, performance, and brand.
4. Synthesis into a short list or final recommendation.
5. Final gate by the orchestrator.

## Hero Evaluation Rubric
Score each idea from 1 to 5:
- mobile visibility
- message clarity
- immersive quality
- performance cost
- accessibility
- brand coherence
- implementation risk

Prefer the highest balance of clarity and safety.

## Synthesis Template


## Anti-Patterns
- Do not run all specialists on every request.
- Do not let style outrun structure.
- Do not add whimsy before the system is usable.
- Do not approve concepts that fail mobile clarity or performance.
- Do not return a long list without a recommendation.
